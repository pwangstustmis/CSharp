物件導向程式簡介

結構化程式設計將原本問題切割成許多可解決小問題，之後再將這些已解決小問題組合起來用於解決原來問題。而物件導向程式不同於結構化程式，它不藉由函數或程式模組來解決問題，而是定義所謂的物件(object)，利用這些物件與物件的互動來解決問題。學習物件導向程式設計，必須要了解下列的重要概念 類別(class)、物件(object)、繼承(inheritance)、抽象(abstraction)、封裝(encapsulation)、多型(polymorphism)，以下將對這些概念簡單說明再進行介紹。

  • 類別(class)：類別用於描述定義不同獨立存在的個體，而這些個體在物件導向程式設計中稱為物件，用於描述定義不同獨立存在的個體，類別一般會包含個體共同的特性(屬性)與行為(方法)，例如簡單射擊遊戲中玩家類別共同特性包含如 圖像、血量、子彈數目、位置 等，行為包含如 射擊、移動、武器更新 等。
  • 物件(object)：類別為物件的抽象定義，物件為類別實際的存在。例如：玩家類別用於描述定義玩家應有特性及對應的行為，當實際程式執行時，C#使用 new 命令建利玩家類別的對應玩家物件，之後玩家物件可實際對應至遊戲中不同的玩家。
  • 繼承(inheritance)：類別具有繼承的概念，新類別可以繼承現有的類別並用於建立新的類別，例如：超級玩家類別除包含現有玩家的設定，還加上新的設定與功能，要設計這樣的超級玩家，我們讓超級玩家繼承玩家類別後再加入新設定的設定與功能。
  • 抽象(abstraction)：抽象的目的是用簡單的事物來表示複雜的動作，其目的用於隱藏抽象概念執行細節，例如我們都知道如何打開電視而不需要知道電視如何被開啟的實際步驟，這種引隱藏執行細節的概念就是抽象。透過抽象概念，使用者只需要知道介面的操作，這讓設計者可以更專注於設計不同或更複雜的步驟。
  • 封裝(encapsulation)：封裝目的為隱藏物件的內部狀態和功能，讓使用者透過允許的公用的函數來存取類別資料與其他功能。
  • 多型(polymorphism)：多型目的是希望能以相同的方式來執行不同的動作，例如動物都會叫，可是叫的動作會不一樣，透過多型的概念，我們就可以設計適涵蓋多種動物叫的動作。
類別(class)定義

類別為一抽象對象的描述，類別描述對象的屬性與動作，如果設計者完整了解描述對象時，可用單一類別來描述對象。當設計者部分了解或不了解描述對象時，可以透過抽象描述abstract、interface來定義類別，之後再對這些動作進行實際製作。因為類別具有繼承功能，可以讓設計者能不斷擴展原先設計藍圖來因應後續新增的功能與變化。

物件導向程式利用物件來解決問題，而類別(class)為物件的抽象描述，類別描述並定義物件的屬性與方法，所以 類別為物件的定義，而物件為類別的實現，舉例來說：建造房屋的藍圖就像類別，而藍圖的實現就是房屋。類別定義包含類別名稱、屬性、方法，屬性是用於描述對應類別特性，而方法是定義類別可執行的動作。

下列程式定義C#類別架構，using用於宣告使用程式模組命名空間，System命名空間包含主控台應用的常用函數定義， namespace 用於設定類別所屬命名空間。命名空間用於定義相近的類別與屬性所屬範圍，同時也可以避免專案整合時會產生名稱衝突的問題。類別定義如自訂類別，定義方式為 存取修飾 class 類別名稱，存取修飾用於設定對象可被存取範圍，常見存取修飾包含public、private、protected、internal，如果類別宣告未設定存取修飾，C#會將存取修飾預設為internal，以Program類別為例，程式定義存取修飾為internal的Program類別，Program類別中Main方法是專案程式執行的進入點，請注意：C#程式只能有一個Main方法，同時Main方法的存取修飾為static，Main方法可由主控台代入參數至變數args。

如先前說明，存取修飾(access modifier)用於設定可被存取的範圍，常見存取修飾包含 public、private、protected、internal，存取修飾 public 定義為公用，可被自己與其他的類別所使用，存取修飾 private 定義為私有，只能給自己的類別使用，存取修飾 protected 定義保護，可以給自己或繼承自己的後代類別所使用，存取修飾 internal 定義內部，可讓同一專案下類別使用，類別屬性與方法如未宣告存取修飾，預設為 private。類別名稱建議使用PascalCase方式命名、屬性、方法的命名方式建議lowerCamel方式命名。

//Program.cs using System; namespace [設計者設定類別空間名稱] { //主程式(專案進入點) class Program { static void Main(string[] args) { Console.WriteLine("Hello World!"); } } //自訂類別 存取修飾 class 類別名稱 { 存取修飾 資料類型 屬性變數名稱; 存取修飾 傳回類型 方法名稱(資料類型 代入參數名稱) { return 傳回值; // } } }

存取修飾(Access Modifier)

如簡介說明，類別為物件的抽象定義，物件為類別實際的存在，要在程式使用對應類別，必須建立對應類別物件，使用 new 類別名稱()可產生對應類別物件，詳細物件建立方式會在物件單元詳述，要能正確地使用對應類別物件屬性與方法，正確存取修飾設定非常重要，以下列程式為例，主程式中s.name="甲"會產生錯誤，因為學生類別中未設定name屬性存取修飾，預設為private，private存取修飾宣告該對象只能在物件本身被存取，不能在本身之外被存取，所以Student物件s的name屬性無法在Program類別中被設定，因此產生錯誤。

//Program.cs using System; namespace myclasses { //主程式(專案進入點) class Program { static void Main(string[] args) { Student s=new Student(); s.name="甲"; //產生錯誤 } } //自訂類別 class Student { string? name; string? address; string? phone; bool Login() { return false; } void Register() { } } }

類別-列舉類別

列舉類別用於宣告一串不可變動常數的組合，設計者透過 enum 命令宣告列舉類別，列舉類別也可宣告於其他類別中，使用列舉類別僅需使用 類別.列舉值 即可傳回類舉類別。

using System; namespace classes { enum Dept { 資訊管理系, 企業管理系, 財務金融系 } class Program { static void Main(string[] args) { Dept d = Dept.資訊管理系; Console.WriteLine(d); } } }

類別屬性存取時可透過 get、set 設定存取的程式區塊，設定方式如下。一般屬性值設定均置於一私有暫時變數中，並利用此私有暫時變數來儲存屬性值，透過此種方式可隱藏真正屬性欄位名稱。

存取修飾 class 類別名稱 { 存取修飾 屬性類型 屬性名稱 { get; set; } } 存取修飾 class 類別名稱 { private 屬性類型 _屬性名稱=起始值; 存取修飾 屬性類型 屬性名稱 { get => _屬性名稱; set => _屬性名稱=value; } }

物件(object)

物件為類別實際的存在，在C#透過 new 命令可以建立對應類別的物件。類別定義中可以包含一個特殊方法稱為建構式(constructor)，建構式為一方法，不具傳回類型、名稱與類別相同、可以有代入值，建構式本身會在建立物件時被自動執行。

// Student類別建構式 using System; namespace classes { class Student { public Student() { Console.Write("從Student類別建構式"); } } class Program { static void Main(string[] args) { Student s=new Student(); //建立物件 } } }

保留字this用於代表類別現在的物件，如果建構式代入參數名稱與屬性名稱相同，程式將無法分辨屬性與代入參數，透過保留字this用於指定物件屬性。

// Student類別建構式 using System; namespace classes { private String Id; public Student(String Id) { Id=Id; //產生錯誤 } class Program { static void Main(string[] args) { Student s=new Student(); //建立物件 } } }
練習
1. 請在上述Student類別加入_name屬性，並設定起始值為 甲 ，之後在建構式中輸出學生姓名。 2. 請在上述Student類別加入_name屬性，在建構式代入姓名並輸出。 3. 請將上述Student類別中_name屬性設為私有，加入公有屬性 name並搭配屬性存取(get、set)，在建構式代入姓名後，在主程式輸出學生姓名。
static保留字

保留字static(靜態)用於宣告共有類別、方法、屬性，當一個類別包含靜態方法、屬性，不需要建立物件久就可以使用這些屬性或方法，使用方式為類別名稱.靜態屬性、類別名稱.方法。請注意：如果類別宣告為static，則該類別不能建立物件。

// Student類別建構式 using System; namespace classes { class Student { public static String SchoolName="南臺科技大學"; } class Program { static void Main(string[] args) { Console.WriteLine(Student.SchoolName); } } }
繼承(inheratance)

繼承(inheritance)讓程式可以重複使用(子類別繼承父類別)屬性與方法、新增自己的屬性與方法、覆寫繼承方法，C#中透過子類別:父類別用於宣告繼承父類別的子類別，子類別會繼承父類別private存取修飾以外的設定，子類別繼承的方法可藉由override宣告來覆寫富類別的方法，但是父類別中對應方法必須宣告為virtual，覆寫的概念會在多型(polymorphism)中介紹。

// Animal類別為父類別，Dog類別繼承Animal類別並新增Display方法 using System; namespace classes { class Animal { public String Name; } class Dog:Animal { public void Display() { Console.WriteLine("動物:"+this.Name); } } class Program { static void Main(string[] args) { Dog d=new Dog(); d.Name="柴犬"; d.Display(); } } }
// Animal類別為父類別，Dog類別繼承Animal類別並新增Display方法 using System; namespace classes { class Animal { public String Name; public virtual void Display() { Console.WriteLine("動物:"); } } class Dog:Animal { public override void Display() { Console.WriteLine("Dog動物:"+this.Name); } } class Program { static void Main(string[] args) { Dog d=new Dog(); d.Name="柴犬"; d.Display(); } } }
練習

抽象(Abstraction)

抽象的概念是用簡單的事物來執行複雜的動作，其目的用於隱藏抽象概念執行細節。在C#中定義抽象概念包含兩個關鍵字abstract、interface，關鍵字abstract用於定義抽象 類別或方法，abstract抽象類別僅能被繼承，當設計類別方法不了解實際如何製作，可宣告這個類別為abstract抽象類別，透過繼承並進行覆寫(override)完成未完成之方法。

public abstract class MobilePhone { public void Call() { Console.WriteLine("撥電話"); } public void Sms() { Console.WriteLine("簡訊"); } public abstract void Color(); public abstract void ScreenSize(); public abstract void OperatingSystem(); } public class iPhone : MobilePhone { public override void Color() { Console.WriteLine("iPhone顏色"); } public override void ScreenSize() { Console.WriteLine("iPhone大小"); } public override void OperatingSystem() { Console.WriteLine("iPhone作業系統"); } }

關鍵字interface用於定義抽象方法，不設定存取修飾、預設為共有(public)、透過繼承並完成未完成之方法。繼承類別為完全繼承包含屬性與方法，如果某些方法僅限定於特殊類別，則可透過interface定義來繼承該方法。

interface MobilePhone { void Color(); void ScreenSize(); void OperatingSystem(); } public class iPhone : MobilePhone { public void Color() { Console.WriteLine("iPhone顏色"); } public void ScreenSize() { Console.WriteLine("iPhone大小"); } public void OperatingSystem() { Console.WriteLine("iPhone作業系統"); } }
多型(Polymorphism)

多型是指他可以允許類別中具有名稱相同但型態不同的方法，多型介紹將涵蓋單一類別多型(overload,多載)與繼承類別多型包含(方法隱藏與override,覆寫)。

同一類別中方法名稱相同、引數或返回值不同的多個方法即為多載(overload)。

// Animal類別建構式多載 using System; namespace classes { class Animal { public String name; public Animal() { Name="無"; } public Animal(string _name) { name=_name; } } class Program { static void Main(string[] args) { Animal a1=new Animal(); } } }

方法隱藏(method hiding)讓子類別可藉由new命令重新定義與父類別相同名稱的函數，子類別呼叫該函數時，是呼叫子類別定義函數，但是子類別可以透過base命令來呼叫父類別的函數。

// Dog類別繼承Animal類別並隱藏Roar方法 using System; namespace classes { class Animal { public void Roar() { Console.WriteLine("無"); } } class Dog:Animal { public new void Roar() { base.Roar(); //透過base呼叫Dog父類別Animal的Roar方法 Console.WriteLine("汪"); } } class Program { static void Main(string[] args) { Dog d=new Dog(); d.Roar(); } } }

覆寫(override)讓子類別繼承父類別函數進行覆蓋，請注意：子類別的函數名稱與參數必須與父類別相同，同時父類別的函數必須加上virtual宣告，子類別函數必須加上override宣告。如果需執行以複寫父類別方法，可使用base關鍵字，如需執行父類別建構式也可透過base命令來呼叫。

// Animal類別為父類別，Dog類別繼承Animal類別並覆寫Display方法 using System; namespace classes { class Animal { public String Name; public virtual void Roar() { Console.WriteLine("無"); } public Animal(string _name) { Name=_name; } } class Dog:Animal { public Dog(string _name) : base(_name) { } public override void Roar() { base.Roar(); Console.WriteLine("汪"); } } class Program { static void Main(string[] args) { Dog d=new Dog("柴犬"); } } }
