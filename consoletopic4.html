物件導向程式- Object Oriented Programming

傳統結構化程式將解決問題的方法切割成許多步驟，接下來將這些步驟撰寫為函數並將這些函數一個一個串接來解決問題。不同於結構化程式，物件導向程式不藉由函數或程式模組來解決問題，而是定義所謂的物件(object)，之後利用這些物件與物件的互動來解決問題。學習物件導向程式設計，大家必須要了解下列的重要概念 類別(class)、物件(object)、繼承(inheritance)、抽象(abstraction)、封裝(encapsulation)、多型(polymorphism)，以下將對這些概念簡單說明再進行介紹。

  • 類別(class)：用於描述定義不同獨立存在的個體，類別會包含個體共同的特性(屬性)與行為(方法)，例如學生類別包含共同特性如 學號、姓名、電話等，行為包含如 資料更新等。
  • 物件(object)：類別為個體的抽象定義，物件為這些類別實際的存在。例如學生類別用於描述學生應有特性及對應的行為，在C#透過 new 命令可以建立對應類別的個體也就是物件，物件可用於設定特定學生資料。
  • 繼承(inheritance)：類別具有繼承的概念，新類別可以繼承現有的抽象概念類別並建立新的抽象概念類別，透過這種機制可延續先前類別設定方法。
  • 抽象(abstraction)：抽象的概念是用簡單的事物來執行複雜的動作，其目的用於隱藏抽象概念執行細節，例如我們都知道如何打開電視而不需要知道電視如何被開啟的實際步驟，這種隱藏複雜步驟的概念就稱為抽象。另外以咖啡機為例，我們只須點選喜歡咖啡種類，並不需要知道實際上咖啡機如何製作一杯咖啡，這種引隱藏執行細節的概念就是抽象。透過抽象概念，只用者只需要知道操作的介面，這個讓設計者可以更專注於設計不同或更複雜的步驟。
  • 封裝(encapsulation)：隱藏物件的內部狀態和功能，並只允許透過公用的函式存取類別資料與其他功能。
  • 多型(polymorphism)：能夠以相同的方式來執行不同的動作，例如動物都會叫，透過物件導向多型的概念，雖然每種動物都會叫，可是叫的動作會不一樣。

類別(class)

重點提示：不同於功能性程式設計(functional programming)，類別為描述對象的整合體，他會包含描述對象的屬性與對應的動作，當設計者完整知道如何規劃與設計描述對象時，可用單一類別來描述對象。當使用者部分了解或不了解描述對象的動作時，可以透過抽象描述abstract、interface來定義類別，之後再對這些動作進行實際製作。同時類別具有繼承功能，讓設計者能不斷擴展原先設計藍圖來因應後續新增的功能與變化。

我們知道物件導向程式利用物件來解決問題，而類別(class)為物件的抽象描述，他描述並定義物件的屬性與方法，使用者可以想像 類別為物件的定義，而物件為類別的實現，另外使用者可以想像：類別就像建造房屋的藍圖，而房屋就是藍圖的實現。類別定義包含類別名稱、屬性、方法，屬性是用於描述對應類別特性，而方法是定義類別可執行的動作。
下列程式中 using 用於宣告程式使用 System 命名空間、下一行 namespace 用於定義程式本身命名空間。命名空間用於定義相近的類別與屬性，同時也可以避免專案整合時會產生名稱衝突的問題。class Program用於定義Program類別，而Program類別中Main方法是專案程式執行的進入點，C#程式只能有一個Main方法，Main方法可由主控台代入參數至變數args。 


using System;
namespace classes
{
  class Program
  {
    static void Main(string[] args)
    {
      Console.WriteLine("Hello World!");
    }
  }
}

定義C#類別、屬性、方法會包含 存取修飾(access modifier) ，存取修飾用於設定該項目可被存取的範圍，常見存取修飾包含 public、private、protected、internal，存取修飾 public 定義可為自己與其他的類別使用，存取修飾 private 定義只能給自己的類別使用，存取修飾 protected 定義可以給自己或繼承自己的後代類別所使用，存取修飾 internal 定義在同一專案下類別使用。類別名稱、屬性、方法的命名方式：private欄位使用 _lowerCamel方式，區域變數及代入參數使用 lowerCamel方式命名，其他使用PascalCase方式命名。 


//未宣告存取修飾類別 預設為internal,未宣告存取修飾屬性與方法 預設為private
//下列程式宣告兩個 internal類別 Student, Program
//類別Student包含公共(public)屬性 Name與方法 Print
using System;
namespace classes {
  class Student {
    public String Name="甲";
    public void Print() {
      Console.Write("從Student類別輸出");

  }

類別-列舉類別

列舉類別用於宣告一串不可變動常數的組合，設計者透過 enum 命令宣告列舉類別，列舉類別也可宣告於其他類別中，使用列舉類別僅需使用 類別.列舉值 即可傳回類舉類別。 


using System;
namespace classes
{
    enum Dept
    {
        資訊管理系,
        企業管理系,
        財務金融系
    }
    class Program
    {
        static void Main(string[] args)
        {
            Dept d = Dept.資訊管理系;
            Console.WriteLine(d);
        }
    }
}

物件(object)

物件為類別實際的存在，在C#透過 new 命令可以建立對應類別的物件。

類別建構式-constructor

建構式為一方法，名稱與類別相同，他會在產生物件時被執行。建構式可以代入值用於設定對應類別物件的起始屬性值。建構式需宣告存取修飾，建構式可以設定代入值。 


// Student類別建構式
using System;
namespace classes {
  class Student {
    public Student() {
      Console.Write("從Student類別建構式");
    }
  }
  class Program {
    static void Main(string[] args) {
      Student s=new Student(); //建立物件
    }
  }
}
p> 上述程式在Main方法建立一學生類別物件s，在產生物件s同時會執行物件s的建構式並輸出從Student類別建構式。 


// Student類別建構式有代入值
using System;
namespace classes {
  class Student {
    public string name;
    public Student(string _name) {
      name=_name;
    }
  }
  class Program {
    static void Main(string[] args) {
      Student s=new Student("John"); //建立物件
      Console.WriteLine(s.name);
    }
  }
}

上述程式在Main方法建立一學生類別物件s，在產生物件s同時在建立物件s時將起始姓名代入建構式中。

保留字this用於代表類別現在的物件，以下列程式為例，如果建構式代入參數名稱與屬性名稱相同，程式無法分辨屬性與代入參數，透過修改建構式為this.Id=Id; 就可以。

保留字- static

保留字static(靜態)用於宣告共有類別、方法、屬性，當一個類別包含靜態方法、屬性，他不需要建立物件久就可以使用這些屬性或方法，使用方式為類別名稱.靜態屬性、類別名稱.方法。請注意：如果類別宣告為static，則該類別不能建立物件。 


// Student類別宣告靜態變數SchoolName
using System;
namespace classes {
  class Student {
    public static String SchoolName="南臺科技大學";
  }
  class Program {
    static void Main(string[] args) {
      Console.WriteLine(Student.SchoolName);
    }
  }
}

練習：請在上述Student類別加入Name屬性，並利用建構式代入學生姓名，比較輸出學生姓名與校名差異。

抽象(Abstraction)

抽象的概念是用簡單的事物來執行複雜的動作，其目的用於隱藏抽象概念執行細節。在C#中定義抽象概念包含兩個關鍵字abstract、interface，關鍵字abstract用於定義抽象 類別或方法，同時abstract抽象類別僅能被繼承，當設計者對類別方法某些部分了解如何製作，而某些部分不了解如何製作，則可宣告這個類別為abstract抽象類別，透過繼承並進行覆寫(override)完成未完成之方法。 


public abstract class MobilePhone
{
  public void Call()
  {
    Console.WriteLine("撥電話");
  }
  public void Sms()
  {
    Console.WriteLine("簡訊");
  }
  public abstract void Color();
  public abstract void ScreenSize();
  public abstract void OperatingSystem();
}
public class iPhone : MobilePhone
{
  public override void Color()
  {
    Console.WriteLine("iPhone顏色");
  }
  public override void ScreenSize()
  {
    Console.WriteLine("iPhone大小");
  }
  public override void OperatingSystem()
  {
    Console.WriteLine("iPhone作業系統");
  }
}

關鍵字interface用於定義抽象 方法，不設定存取修飾，預設為共有(public)，透過繼承並完成未完成之方法。繼承類別為完全繼承包含屬性與方法，如果某些方法僅限定於特殊類別，則可透過interface定義來繼承該方法。 


interface MobilePhone
{
  void Color();
  void ScreenSize();
  void OperatingSystem();
}
public class iPhone : MobilePhone
{
  public void Color()
  {
    Console.WriteLine("iPhone顏色");
  }
  public void ScreenSize()
  {
    Console.WriteLine("iPhone大小");
  }
  public void OperatingSystem()
  {
    Console.WriteLine("iPhone作業系統");
  }
}

封裝(Encapsulation)

隱藏物件的內部狀態和功能，並只允許透過公用的函式存取類別資料與其他功能。藉由宣告存取修飾詞來隱藏不希望被使用者直接存取內容稱為封裝。

多型(Polymorphism)

多型是指他可以允許類別中具有名稱相同但型態不同的函數，多型介紹單一類別多型(overload,過載)與繼承類別多型包含(方法隱藏與override)。

  • overload(過載)：在同一類別中方法名稱相同、引數或返回值不同的多個方法即為過載(overload)。 
    •

    class Dog:Animal {
      public new void Roar() {
        base.Roar();           //透過base呼叫Dog父類別Animal的Roar方法
        Console.WriteLine("汪");
      }
    }

    class Program {
      static void Main(string[] args) {
        Dog d=new Dog();
        d.Roar();
      }      
    }
  }
  • override(覆寫)：對子類別繼承父類別函數進行覆寫，請注意：在子類別的函數名稱與參數必須與父類別相同，同時父類別的函數必須加上virtual宣告，子類別函數必須加上override宣告。建構式可透過base命令來呼叫。 
    • 
// Animal類別為父類別，Dog類別繼承Animal類別並覆寫Display方法
  using System;
  namespace classes {
    class Animal {
      public String Name;
      public virtual void Roar() {
        Console.WriteLine("無");
      }
      public Animal(string _name) {
        Name=_name;
      }
    }
    class Dog:Animal {
      public Dog(string _name) : base(_name)
      {
      }
      public override void Roar() {
        Console.WriteLine("汪");
      }
    }

    class Program {
      static void Main(string[] args) {
        Dog d=new Dog("柴犬");
      }     
    }
  }
